## React Analysis

### Hooks Analysis
- **useEffect dependencies**: Are all dependencies included? Are there stale closures?
- **useCallback/useMemo**: Are they necessary? Are dependencies correct?
- **useRef**: Is it used correctly for mutable values vs DOM refs?
- **Custom hooks**: Do they follow the rules of hooks?

### State Management
- **Stale state**: Are there race conditions in state updates?
- **Batching issues**: Are multiple state updates handled correctly?
- **Unnecessary re-renders**: Are memo/callback used appropriately?
- **Prop drilling**: Would context be more appropriate?

### Component Patterns
- **Controlled vs uncontrolled**: Are form inputs handled correctly?
- **Error boundaries**: Are errors caught and handled gracefully?
- **Cleanup**: Are effects cleaned up properly (subscriptions, timers)?
- **Memory leaks**: Are async operations cancelled on unmount?

### Performance
- **Re-render optimization**: React.memo, useMemo, useCallback usage
- **Bundle size**: Large imports that could be code-split
- **List rendering**: Proper keys, virtualization for long lists

## TypeScript Analysis

### Type Safety
- **any usage**: Should be replaced with proper types
- **Type assertions**: Are they safe or hiding potential issues?
- **Union types**: Are all cases handled in switch/if statements?
- **Generic constraints**: Are they correct and useful?

### Common Patterns
- **Optional chaining**: Is ?. used appropriately?
- **Nullish coalescing**: Is ?? vs || used correctly?
- **Type guards**: Are they narrowing types correctly?
- **Discriminated unions**: Are they used for complex state?

### API Types
- **API responses**: Are they typed?
- **Error types**: Are errors properly typed?
- **Form types**: Are form values typed?
