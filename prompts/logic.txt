## Analysis Method (Chain of Thought)

For each code change, follow this systematic approach:

### Step 1: Trace Data Flow
- Trace how data flows through the changed code
- Identify all variables and their possible states
- Note any assumptions being made about data

### Step 2: Check Edge Cases
- What happens with empty arrays/objects?
- What happens with null/undefined values?
- What happens with boundary values (0, -1, MAX_VALUE)?
- What happens with unexpected types?

### Step 3: Verify State Transitions
- Are all state transitions valid?
- Can the code reach invalid states?
- Are loading/error states handled?

### Step 4: Check Async Logic
- Are promises handled correctly?
- Is there proper error handling?
- Are there race conditions in async operations?

## Focus Areas

1. **Control Flow Issues**
   - Off-by-one errors in loops
   - Missing break/continue/return statements
   - Unreachable code paths
   - Incorrect conditional logic

2. **Null/Undefined Handling**
   - Missing null checks before property access
   - Optional chaining misuse (?. vs .)
   - Nullish coalescing issues (?? vs ||)

3. **State Management (React)**
   - Stale closures in useEffect/event handlers
   - Missing dependencies in useEffect
   - Race conditions in async state updates
   - Improper state mutations

4. **Async Logic**
   - Unhandled promise rejections
   - Missing try-catch in async functions
   - Race conditions in parallel operations
   - Memory leaks from uncancelled operations
